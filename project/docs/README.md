# Карта Документації

## Призначення

Цей файл є короткою картою документації для `project/`.

Його мета проста: не дати активним документам загубитися серед історичних планів, нотаток рев'ю і підсумкових записів.

## Що Вважати Активними Документами

Якщо потрібно зрозуміти, де зараз істина, починайте звідси.

| Якщо вам потрібно... | Читайте це |
| --- | --- |
| зрозуміти, що це вже не бібліотечний fork, а продуктовий repo | [REPO_STRATEGY.md](../../REPO_STRATEGY.md) |
| зрозуміти, як влаштований сам продукт і куди далі дивитися | [project/README.md](../README.md) |
| зрозуміти стратегічний напрямок: коли і як продукт відокремиться від бібліотечного форка | [PLAN_APP_REPO_EXTRACTION.md](plans/PLAN_APP_REPO_EXTRACTION.md) |
| виконувати поточну фазу розриву залежностей без розмивання scope | [PLAN_APP_DEPENDENCY_DECOUPLING.md](plans/PLAN_APP_DEPENDENCY_DECOUPLING.md) |
| щоденно працювати з workspace об'єкта | [INTAKE_OPERATOR_GUIDE.md](methodology/INTAKE_OPERATOR_GUIDE.md) |
| зрозуміти, що саме зафіксовано в baseline `v1` і де його межі | [V1_CLOSEOUT_2026-04-03.md](reviews/V1_CLOSEOUT_2026-04-03.md) |

## Що Вважати Історичними Документами

У `docs/plans/` і `docs/reviews/` накопичено багато корисного матеріалу, але не весь він має однакову вагу.

Суворе правило таке:

- якщо документ прямо не позначений як активний поточний план або як чинний опис поточної поведінки, вважайте його історичним артефактом;
- нотатки рев'ю, документи повторного рев'ю, старі плани виконання і чернетки багатомодельного зіставлення не керують поточною роботою;
- їх потрібно читати лише тоді, коли потрібна простежуваність або пояснення, чому ми прийшли саме до теперішнього стану.

## Документаційна Дисципліна

Щоб документація знову не перетворилась на кашу, тримаємося таких правил:

- новий активний план має прямо називати, який попередній документ він замінює;
- документ, який більше не веде роботу, не лишається в підвішеному стані, а отримує явний архівний статус;
- чинний опис поточної поведінки не ховається в коротких нотатках рев'ю;
- користувацький робочий шлях не описується через історичні планувальні документи.
